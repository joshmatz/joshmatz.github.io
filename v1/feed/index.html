With the growing presence of mobile devices on the world wide web, it’s becoming ever more necessary to properly test and debug our websites on these new devices.

Fortunately for us, the Chrome team and Apple are working to make it easier and easier.

 

Chrome

Chrome gets points for being useful in development no matter your OS. Google is working hard to cater to developers out there and if you haven’t checked out their developer portal, you should!

Within their portal, you’ll find the Chrome Remote Debugging page that will walk you through the steps you need to take. Now, I’m usually using a Mac and I’m not very familiar with most Terminal commands, so I found this tutorial at Wes Bos to be particularly helpful.

Safari

Unfortunately for developers running Windows or Linux, Apple isn’t so kind to your needs. That said, if you’re running OSX then Apple has made it particularly easy to start debugging from your iOS6 devices. It really is as simple as flicking on a setting on your iDevice.

Internet Explorer

With Microsoft launching Windows 8 recently, being able to debug these devices will also increase in importance. Unfortunately, for now, I don’t have any of these devices to test on. If you have any experience or links to help others out in developing for Microsoft devices, please leave a comment!
